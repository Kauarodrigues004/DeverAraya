Na tag <'input'> com atributo type="text", para que serve o atributo "placeholder"?

Resposta: O atributo placeholder em um <'input type="text"> exibe um texto indicativo dentro do campo, informando ao usuário o que deve ser digitado. Esse texto desaparece quando o usuário começa a digitar.

Na tag <'input'> com atributo type="radio" ou type="checkbox", para que serve o atributo "checked"?

Resposta: O atributo checked em <'input type="radio"''> ou <'input type="checkbox"''> indica que a opção deve estar selecionada por padrão ao carregar a página. Para checkboxes, significa que estarão marcados; para radio buttons, que essa opção estará selecionada.

Na tag <'input'> para que serve o atributo "value"?

Resposta: O atributo value na tag define o valor que será enviado ao servidor quando o formulário for submetido. Para campos de texto, ele também pode exibir um valor padrão dentro do campo.

Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

Resposta: atributo name

Para que serve a tag <'label'>? Para que serve o atributo "for" nessa tag?

Resposta: A tag <'label'> associa um rótulo a um controle de formulário, melhorando a acessibilidade e a usabilidade. O atributo for conecta o rótulo ao controle correspondente, permitindo que o usuário clique no rótulo para selecionar o campo.

Na tag <'option'> para que serve o atributo "selected"?

Resposta: O atributo selected na tag <'option'> indica que a opção deve ser exibida como selecionada por padrão em um menu suspenso. Isso faz com que essa opção apareça como a escolha inicial ao abrir o menu.

>Nas tags de formulário para que serve o atributo "required"?

Resposta: O atributo required indica que um campo de formulário deve ser preenchido obrigatoriamente antes do envio. Se estiver vazio, o formulário não será submetido e o usuário receberá uma mensagem de erro.

Para que serve a tag <'textarea'>?

Resposta: A tag <'textarea'> cria um campo de entrada de texto multilinha em um formulário, permitindo que os usuários insiram textos longos. É ideal para comentários ou descrições, oferecendo mais espaço que um campo de texto simples.

Na tag <'form'> para que serve o atributo "action"?

Resposta: O atributo action na tag <'form'> define a URL para onde os dados do formulário serão enviados quando ele for submetido. Ele determina o destino que processará as informações inseridas no formulário.

Na tag <'form'> para que serve o atributo "method"?

Resposta: atributo method na tag <'form'> define o método HTTP utilizado para enviar os dados do formulário. Os métodos mais comuns são GET, que anexa os dados à URL, e POST, que envia os dados no corpo da requisição.

Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag <'form'>?

Resposta: Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, o valor do atributo method na tag

deve ser POST. Esse método envia os dados no corpo da requisição, em vez de anexá-los à URL.

Qual valor deve ser informado no atributo "method" na tag <'form'> em um formulário que será usado para fazer upload de arquivos?

Resposta: Para fazer upload de arquivos, o atributo method na tag deve ser POST. Além disso, o atributo enctype deve ser definido como multipart/form-data para garantir o envio correto dos arquivos.

O que as tags <'fieldset'> e <'legend'> produzem?

Resposta: As tags <'fieldset'> e <'legend'> são usadas para agrupar e descrever controles relacionados em formulários. O <'fieldset'> cria uma borda ao redor dos elementos, enquanto o <'legend'> fornece um título ou descrição para o grupo.
